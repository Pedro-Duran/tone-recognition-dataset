# Tone Recognition Dataset (THCHS-30)

Pipeline for converting THCHS-30 into a syllable-level Mandarin tone classification dataset.

---

## Project structure

```
tone-recognition-dataset/
├── data/
│   └── raw/                        # place THCHS-30 here (or point --data_dir elsewhere)
├── output/
│   ├── <split>/
│   │   ├── metadata.csv            # Phase 1 output
│   │   ├── metadata.json
│   │   ├── aligned/
│   │   │   ├── metadata_aligned.csv   # Phase 2 output (+ timestamps)
│   │   │   └── metadata_aligned.json
│   │   └── clips/
│   │       ├── tone_1/             # Phase 3 output (syllable .wav files)
│   │       ├── tone_2/
│   │       ├── tone_3/
│   │       ├── tone_4/
│   │       └── tone_5/
├── mfa_corpus/                     # generated by prepare_mfa_corpus.py
├── scripts/
│   ├── inspect_dataset.py
│   ├── generate_tone_metadata.py
│   ├── prepare_mfa_corpus.py
│   ├── parse_textgrids.py
│   ├── slice_audio.py
│   └── utils.py
├── requirements.txt
└── README.md
```

## Installation

### Python dependencies

```bash
pip install -r requirements.txt
```

Python 3.10+ required (uses `str | None` union syntax).

### Montreal Forced Aligner (MFA)

MFA is installed via conda, separately from the Python dependencies above:

```bash
conda create -n aligner -c conda-forge montreal-forced-aligner
conda activate aligner
mfa model download acoustic mandarin_mfa
mfa model download dictionary mandarin_mfa
```

## Phase 1 — Label generation

### Step 1: Inspect the dataset

```bash
python scripts/inspect_dataset.py \
    --data_dir "C:/path/to/data_thchs30/data_thchs30"
```

Prints file counts, split sizes, lexicon entries, and a sample alignment check.

### Step 2: Generate metadata

```bash
# Dev split (recommended to test first)
python scripts/generate_tone_metadata.py \
    --data_dir  "C:/path/to/data_thchs30/data_thchs30/data" \
    --split_dir "C:/path/to/data_thchs30/data_thchs30/dev" \
    --output_dir output/dev/

# Training split
python scripts/generate_tone_metadata.py \
    --data_dir  "C:/path/to/data_thchs30/data_thchs30/data" \
    --split_dir "C:/path/to/data_thchs30/data_thchs30/train" \
    --output_dir output/train/
```

Outputs `metadata.csv` and `metadata.json` — one row per syllable with tone label.

---

## Phase 2 — Forced alignment (MFA)

### Step 3: Prepare MFA corpus

```bash
python scripts/prepare_mfa_corpus.py \
    --data_dir  "C:/path/to/data_thchs30/data_thchs30/data" \
    --split_dir "C:/path/to/data_thchs30/data_thchs30/dev" \
    --corpus_dir mfa_corpus/dev/
```

Creates `mfa_corpus/dev/{speaker}/` directories, each containing:
- `{stem}.wav` — hard-linked from the original (no extra disk space)
- `{stem}.lab` — pinyin tokens from the `.trn` file, space-separated

### Step 4: Run MFA

```bash
conda activate aligner

# Validate corpus before aligning (recommended)
mfa validate mfa_corpus/dev/ mandarin_mfa mandarin_mfa

# Align
mfa align mfa_corpus/dev/ mandarin_mfa mandarin_mfa output/dev/textgrids/
```

MFA writes one `.TextGrid` per utterance to `output/dev/textgrids/`.

### Step 5: Parse TextGrids

```bash
python scripts/parse_textgrids.py \
    --textgrid_dir output/dev/textgrids/ \
    --metadata_csv output/dev/metadata.csv \
    --output_dir   output/dev/aligned/
```

Joins MFA timestamps back into the metadata, adding `start_time` and `end_time` columns.

---

## Phase 3 — Audio slicing

### Step 6: Slice audio into syllable clips

```bash
python scripts/slice_audio.py \
    --aligned_metadata output/dev/aligned/metadata_aligned.csv \
    --clips_dir        output/dev/clips/
```

Cuts each `.wav` at the aligned boundaries and saves one clip per syllable,
organised into `clips/tone_{1..5}/` subdirectories ready for classifier training.

## Output formats

### metadata.csv (Phase 1)

```
sample_id,original_audio_path,syllable_index,hanzi,pinyin,tone
A11_0_0,/data/A11_0.wav,0,绿,lv4,4
A11_0_1,/data/A11_0.wav,1,是,shi4,4
A11_0_2,/data/A11_0.wav,2,阳,yang2,2
```

### metadata_aligned.csv (Phase 2, adds timestamps)

```
sample_id,original_audio_path,syllable_index,hanzi,pinyin,tone,start_time,end_time
A11_0_0,/data/A11_0.wav,0,绿,lv4,4,0.12,0.38
```

### Tone labels

| Value | Meaning              |
|-------|----------------------|
| 1     | First tone (高平)    |
| 2     | Second tone (升)     |
| 3     | Third tone (上)      |
| 4     | Fourth tone (去)     |
| 5     | Neutral tone (轻声)  |

To map neutral tone to `0` instead of `5`, set `NEUTRAL_TONE_AS_ZERO = True`
in [scripts/utils.py](scripts/utils.py).

## Pinyin source

THCHS-30 `.trn` files include a pinyin line (one token per syllable with tone number),
so `pypinyin` is used only as a fallback for any file that lacks it.
